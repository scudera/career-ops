## Visibility Schema

The Visibility field controls which roles appear in which CV variants.

| Value | Behavior |
|---|---|
| `always` | Include in every variant |
| `variant-specific` | Include only if target variant is in the role's `Variants` list |
| `on-request` | Skip unless user explicitly requests this role |
| `reference-only` | Never include in output (alias/metadata only) |

### Known variants

| Variant | Target market | Notes |
|---|---|---|
| `eu` | Pan-EU / remote EU | Default for EU job applications |
| `uk` | United Kingdom | Brexit visa caveat; emphasize EU experience pre-2020 less |
| `ie` | Ireland | EU-equivalent; cidadania PT cobre |
| `pt` | Portugal | Local market; PT-language CV |
| `br` | Brasil | PT-BR; local subsidiária pharma |
| `us` | United States | Less emphasis (visa risk) |

## How to use this file (instructions for tailoring)

1. **Set up:** copy this file to `docs/resume/master-profile.md` and edit with real data.
2. **Generate variant:** `node scripts/generate-cv-variant.mjs docs/resume/master-profile.md --variant=eu --output=cv-eu.md`
3. **Iterate:** after each application, update master profile with any
   new role facts/achievements discovered during JD analysis.
4. **Privacy:** the real `docs/resume/master-profile.md` is gitignored; never commit.
